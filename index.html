Xzy запоминает архитектурные решения команды и предупреждает AI-ассистента, когда новый запрос им противоречит.

Интеграция через MCP-сервер и Hooks в Claude Code и Cursor

Как это выглядит в чате

Когда вы просите модель добавить Redis, Xzy перехватывает намерение на уровне протокола MCP, находит прошлое решение (ADR-42) и показывает предупреждение:

> Добавим Redis для хранения сессий пользователей.
Xzy Memory Flag (Приоритет: P2 / Архитектура)

Запрос противоречит зафиксированному техническому решению (ADR-42).

Согласованное решение: Отказ от Redis для сессий (Март).
Обоснование: Увеличивал задержки (latency) на критическом пути. Переход на stateless JWT.
Критерий пересмотра (override_condition): Нагрузка превысит 5000 RPS и потребуется мгновенный отзыв токенов.

Требуется подтверждение для обхода ограничения и фиксации технического долга.

Три уровня контроля на каждом запросе

P1

Агент-Безопасник

Статический анализ кода (AST/regex). Ловит критические уязвимости (SQLi, eval, секреты в коде) без вызова LLM. Спорит жестко и требует явный override.

P2

Агент-Память

Быстрый фильтр по тегам технологий + семантический поиск в базе ADR. Находит архитектурные конфликты с прошлыми решениями вашей команды.

P3

Агент-Стилист

Проверяет код на соответствие гайдлайнам оформления. Молча предлагает автофикс (autofix) для мелких правок, не отвлекая разработчика.

Как Xzy обрабатывает каждый запрос за 5 шагов:

  1. Захват: Система перехватывает промпт и определяет, содержит ли он архитектурные намерения.
  2. Теговый фильтр: Дешевое сравнение по ключевым словам и технологиям для отсечения лишнего.
  3. Семантический поиск: Поиск по смыслу среди top-5 похожих записей в базе архитектурных решений.
  4. Анализ конфликта: LLM сравнивает намерение разработчика с найденным ADR и определяет тип пересечения.
  5. Фиксация: Решение сохраняется. В случае обхода — записывается в базу как долг с меткой OVERRIDDEN.

Честные ограничения Xzy

Не заменяет контекст сессии

Xzy не решает проблему потери нити разговора внутри одной сессии (это задача Cursor/Claude Code). Мы решаем проблему потери контекста между сессиями во времени.

Не для соло-разработчиков на один вечер

Если проект живет пару недель, Xzy не окупит себя. Инструмент раскрывает ценность в командах, где люди сменяются, решения забываются, а проект живет месяцами.
